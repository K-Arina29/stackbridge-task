# Задание 3. Архитектура PUSH-уведомлений

## Описание
Интернет-магазин «Петрушка Зелёная» планирует внедрить push-уведомления в мобильное приложение для повышения вовлечённости пользователей (статус заказа, акции, отмена, напоминания и т.д.).

## Высокоуровневая схема архитектуры

Мобильное приложение  
→ (device token)  
Backend API  
→ (сохранение)  
База данных  
→  
Message Broker (RabbitMQ / Kafka)  
→  
Push Notification Worker  
→ FCM (Android)  
→ APNs (iOS)

## Как это работает

1. Пользователь разрешает push-уведомления в приложении.
2. Приложение отправляет `device token` на Backend.
3. Backend сохраняет токен в базе данных.
4. Когда происходит событие (например, изменение статуса заказа), сообщение отправляется в Message Broker.
5. Push Worker забирает сообщение и отправляет уведомление через FCM (Android) или APNs (iOS).
